# 架构说明

## 模块分层

- `routers`：提供 REST API，保持接口字段贴近前端页面需求。
- `services`：封装 JD 解析、去重、新岗位指数、岗位能力更新、幻觉防控、简历解析和匹配分析。
- `models`：SQLAlchemy ORM 模型，覆盖数据源、原始 JD、解析 JD、岗位、技能、图谱关系、更新事件、简历、匹配报告、审核任务和测试用例。
- `db/init_db.py`：生成系统演示与测试所需的种子数据。

## AI 接入边界

`services/ai_provider.py` 统一封装 JD 解析、简历解析、人岗匹配、学习路径和数字人面试官的 AI 调用入口。系统保留本地模型适配层与外部大模型接入边界，便于在不改变前后端业务流程的情况下切换模型服务。

## 幻觉防控

`services/hallucination_guard.py` 检查：

- 是否带有 `evidence`
- 置信度是否低于阈值

低置信度内容会写入 `ReviewTask`，由人工审核页面处理。

## 匹配公式

总匹配度 = 必备技能匹配度 * 0.4 + 加分技能匹配度 * 0.15 + 项目经验匹配度 * 0.2 + 工具平台匹配度 * 0.1 + 行业场景匹配度 * 0.1 + 证书成果匹配度 * 0.05

## 新岗位指数

新岗位指数 = 技能增长率 * 0.3 + 多源一致性 * 0.25 + 技能组合新颖度 * 0.25 + 岗位标题稳定性 * 0.1 + 行业场景扩散度 * 0.1
